- Markdown adapter 第一版使用语法感知扫描器保护关键结构；后续可在同一 adapter 边界内替换为完整 AST parser。
- VS Code 插件默认 provider 暂设为 `fake`，便于首次运行；用户可切换为 `openai-compatible` 并通过命令写入 API key。
- 设计文档 settings 示例同步为当前实现默认值，并注明发布前需要调整真实 provider 选择流程。
- AI 分段预算由 provider 能力表达，OpenAI-compatible 默认 `maxContextTokens=128000`、`maxOutputTokens=4096`，可通过 CLI 和 VS Code 设置面板配置。AI 请求区分 `referenceDocument.units` 和 `translationUnitIds`，目标范围受输出 token 和最小输入 payload 限制，参考上下文会在剩余输入预算内按滑动窗口扩展。
- API key 均通过环境变量、CLI 参数或 VS Code SecretStorage 注入，不写入 metadata。

## Phase 2：AI 双预算分段收尾

时间：2026-07-14
Agent：Codex
项目根：`D:\Users\g-tty\Documents\VSCode_Doc_Translator`

### 目标

按用户补充要求，将 AI 大文件分段从“最大上下文字符”推进为“模型最大上下文 token + 单次最大输出 token”的双预算模型，并同步 VS Code 设置面板、CLI、文档和测试语义。

### 变更

- OpenAI-compatible provider 的大文件分段以 `maxContextTokens` 和 `maxOutputTokens` 为主；`maxOutputTokens` 同时限制本次目标 translation ids 的预计输出，并作为 API `max_tokens` 发送。
- 可用输入预算按 `maxContextTokens - maxOutputTokens - promptOverheadTokens` 近似计算；目标范围确定后，`referenceDocument.units` 在剩余输入预算内围绕目标范围滑动扩展。
- `maxContextCharacters` 仍保留为兼容旧 CLI/env/config 的近似换算入口，但不再是首选配置模型。
- VS Code Settings webview 暴露模型上下文 token 和最大输出 token，避免用户直接编辑 settings JSON。
- `README.md`、`docs/design.md`、`docs/architecture.md`、ADR 和 `AGENTS.md` 已同步新合同；其中 `docs/architecture.md` 因原文件乱码，已重写为可读架构说明。

### 验证

- `npm run check`：通过，6 个测试文件、13 个测试全部通过。
- `npm audit --omit=optional`：通过，0 vulnerabilities。

### 风险 / TODO

- Token 估算仍使用字符数近似，不等同于 provider 的真实 tokenizer。
- 单个超长 translation unit 仍可能超过输入或输出预算；当前会单独请求并记录 warning，后续需要更细 unit 切分。
- OpenAI-compatible live API 行为仍需用户提供 endpoint/model/API key 后验证。

### 目标

完成可选 hidden output mode，并明确 VS Code 点击翻译时的缓存行为：源 hash 未变则复用已翻译文档，源 hash 改变则重新翻译。

### 变更

- 新增 core 输出模式 `same-dir | hidden-cache`，默认仍为 `same-dir`。
- `hidden-cache` 会把译文文件写入源目录旁的 `.vscode-doc-translator-cache/`，metadata 仍写在同一隐藏目录中，并记录 `target.directoryMode`。
- 缓存 fresh 判断增加输出模式匹配；source hash、目标语言、provider、输出模式和译文 hash 都匹配时才复用缓存。
- CLI 新增 `--output same-dir|hidden-cache`。
- VS Code 新增 `docTranslator.output.directoryMode` 配置，并在 Doc Translator Settings 面板中提供输出位置下拉框。
- 设置面板保留默认翻译 provider 下拉选择，翻译命令会使用该 provider 执行翻译。
- 补充测试覆盖 hidden-cache 输出、按输出模式复用缓存，以及源文件 hash 改变后重新翻译。
