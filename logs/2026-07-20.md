# 2026-07-20

## Native LLM APIs and public repository preparation

### Goal

Add native OpenAI Responses, Anthropic Messages, and Gemini GenerateContent support, publish multilingual user documentation, package version 0.2.0, and prepare the project for a public GitHub repository.

### Discoveries

- The existing `openai-compatible` adapter called only `/chat/completions`; endpoint replacement alone could not support the three requested protocols.
- OpenAI Responses uses `text.format.json_schema`; Anthropic Messages uses `output_config.format`; Gemini GenerateContent uses `generationConfig.responseJsonSchema`.
- The OpenAI migration guide was re-read through the user-provided local proxy and confirmed `/v1/responses`, `text.format`, typed `output` items, and `store: false` behavior used by the implementation.
- All AI providers can share the existing ordered-context chunking, missing-id repair, and JSON recovery behavior.
- The local repository had no GitHub remote. The `G-TTYg/vscode-doc-translator` repository name was available when checked.
- Three pre-existing untracked `README.auto.*` files remain user-owned and are excluded from the planned commit and VSIX.

### Changes

- Extracted shared structured-LLM workflow into `structuredLlmProvider.ts`.
- Added native OpenAI Responses, Anthropic, and Gemini adapters and protocol-level tests.
- Added provider-specific settings and SecretStorage entries; the visual panel now shows only the selected provider configuration.
- Bumped the extension to 0.2.0 and added GitHub package metadata.
- Rewrote the English marketplace README and added `README.zh-CN.md` plus `CHANGELOG.md`.
- Updated design, architecture, and ADR documentation.
