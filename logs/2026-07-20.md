## Native LLM APIs and public repository preparation

### Goal

Add native OpenAI Responses, Anthropic Messages, and Gemini GenerateContent support, publish multilingual user documentation, package version 0.2.0, and prepare the project for a public GitHub repository.

### Discoveries

- The existing `openai-compatible` adapter called only `/chat/completions`; endpoint replacement alone could not support the three requested protocols.
- OpenAI Responses uses `text.format.json_schema`; Anthropic Messages uses `output_config.format`; Gemini GenerateContent uses `generationConfig.responseJsonSchema`.
- The OpenAI migration guide was re-read through the user-provided local proxy and confirmed `/v1/responses`, `text.format`, typed `output` items, and `store: false` behavior used by the implementation.
- All AI providers can share the existing ordered-context chunking, missing-id repair, and JSON recovery behavior.
- The local repository had no GitHub remote. The `G-TTYg/vscode-doc-translator` repository name was available when checked.
- Three pre-existing untracked `README.auto.*` files remain user-owned and are excluded from the planned commit and VSIX.

### Changes

- Extracted shared structured-LLM workflow into `structuredLlmProvider.ts`.
- Added native OpenAI Responses, Anthropic, and Gemini adapters and protocol-level tests.
- Added provider-specific settings and SecretStorage entries; the visual panel now shows only the selected provider configuration.
- Bumped the extension to 0.2.0 and added GitHub package metadata.
- Rewrote the English marketplace README and added `README.zh-CN.md` plus `CHANGELOG.md`.
- Updated design, architecture, and ADR documentation.

## Success-oriented protected-token recovery

### Discovery

- A live AI translation translated prose but changed or removed protected placeholders across many Markdown units, causing the entire document to fail during final format validation.
- Placeholder names restarted at zero in each unit, so the same token string referred to different protected values in one reference window.

### Changes

- Made protected token names unique by unit order and bumped the shared AI harness to version 3.
- Added local normalization for exact, Markdown-escaped, bold-wrapped, and original-value token forms.
- Added one focused protected-token repair request for affected ids only.
- Changed persistent token failure from a whole-document error to a source-preserving fallback for affected units; the rest of the translation is written normally with warnings.
- Bumped the extension and core to 0.2.2 and the Markdown/MDX adapters to 0.2.0.

### Verification

- Focused AI harness, packaging, chunking, and application tests: passed, 4 files / 22 tests.
- `npm run check`: passed, 9 files / 36 tests.

- VSIX inspection confirmed version `0.2.2`, publisher `vscode-doc-translator`, compiled harness recovery code, both marketplace READMEs, and exclusion of source, tests, logs, generated translations, and secrets.

## Corpus-verified AI translation recovery

### Discovery

- The live OpenAI Responses gateway sometimes returned a flat id map or a single `{ id, text, skip }` object instead of the requested wrapper.
- One missing-id retry could still leave the format adapter without a complete translation map.
- The source-echo gate still rejected a whole document after its focused retry, even when most units were usable.
- Three remaining corpus failures were long Simplified Chinese documents being translated to `zh-CN`; unnecessary API calls exhausted output budgets and produced incomplete responses.

### Changes

- Added response-shape recovery and one malformed-object retry.
- Added bounded missing-id, source-echo, and protected-token fallbacks that preserve only unresolved units and continue document reconstruction.
- Added cross-unit protected-token leak detection and focused repair context.
- Added automatic already-target document detection for AI providers when source language is `auto`.
- Bumped the extension/core to `0.2.3` and AI harness to version 4; updated README files, design, architecture, changelog, and ADRs.

### Verification

- Focused AI provider and harness tests passed after each recovery change.
- Live OpenAI Responses testing used temporary copies of all 34 Markdown files under `MindustryAI/docs` (440,359 source bytes); result: 34 passed, 0 failed.
- `roadmap.md` completed with 7 requests, no source fallback, and no residual harness tokens.
- The three previously failing already-Chinese documents completed locally with zero API requests.
- Source files were never overwritten; all translated artifacts and metadata were written under a temporary directory.

### Risk

- Bounded fallback can leave isolated technical identifiers or unrepaired units in the source language; warnings record each occurrence instead of failing the entire document.
